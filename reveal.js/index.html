  • Inconsistent lighting and shadows

🔍Data Understanding: Diversity is Key

We combined two large Kaggle datasets for high variance.

Deepfake Image Detection Dataset

  • Purpose: Increase diversity beyond just human faces.

DeepDetect-2025

  • Focus: State-of-the-Art Generative Models (StyleGAN3, DALL-E 3, Midjourney).

Result: High diversity, balanced class distribution, and clean data suitable for CNNs.

Model Choice: EfficientNetV2B3

The choice was driven by the trade-off between Accuracy and Computational Efficiency.

  • Justification: Superior balance of speed and accuracy compared to B0 (too shallow) and Xception (too slow).
  • Crucial Feature: Structurally deeper and wider, enabling it to extract fine-grained, high-resolution features critical for detecting subtle artifacts.

Methodology: Mandatory Input Standardization

Problem: Inconsistent crops and background noise lead to failed generalization.

Solution: MTCNN Face Cropping

  • MTCNN (Multi-Task Cascaded CNN) detects the primary face.

  • The face is cropped and resized to a consistent 224x224 input.

  • Result: Forces the model to exclusively focus its capacity on the facial evidence, removing background distraction.

Methodology: Two-Phase Fine-Tuning

Strategy to overcome the initial catastrophic failure of simple transfer learning (B0).

Phase 1: Head Initialization

  • Base Model: Frozen.

  • Goal: Quickly train the new classification layers (head) using LR = 1e-4.

Phase 2: Deep Feature Refinement

  • Base Model: Unfrozen (layers 20% to 100%).

  • Goal: Adjust core features using ultra-low LR = 5e-7.

  • Effect: Specializes the powerful ImageNet knowledge for subtle deepfake artifact detection.

Core Methodology: Triplet Feature Separation

We solve the generalization issue using a Dual-Loss system based on Metric Learning.

  • Goal: Achieve robust feature separation, pushing FAKE features away from REAL clusters.

  • Metric Logic: The model learns based on feature distances (e.g., Triplet Loss concept).

  • Result: Creates a tight, precise decision boundary that is less vulnerable to new types of deepfakes.

Core Finding: Deployment Calibration

  • Problem: The final model exhibited a subtle feature separation failure, where the P(Real) scores for highly manipulative Fakes and true Reals were too close.

  • Final Solution: We implemented a reversed classification rule using a calibrated Threshold of 0.63 to force correct separation in the final application.
